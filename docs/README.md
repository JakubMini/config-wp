# Design documentation

This directory holds the design notes, decision records, and the take-home
writeup for the configuration management exercise.

Files (added over the course of the work):

- `DESIGN.md`  — architecture, on-flash format spec, threading model.
- `GAPS.md`    — known shortcomings from the structured review, with
                 prioritised fixes.

The code lives in `../src/`; the build is driven from the root `CMakeLists.txt`.
